rotations¶

The rotations.py file contains methods that allow a programmer to conduct coordinate frame rotations with dirction cosines as well as quaternians. Presently this library is only set up to handle Tait-Bryan angles.

Euler Angles¶

The following functions will conduct passive rotations based on Direction Cosines.

geo_py.rotations.rotation_matrix(x_rot: float, y_rot: float, z_rot: float, order: str = 'ZYX', deg: bool = False, extrinsic: bool = False) → ndarray[source]¶
Parameters:

  • x_rot – The rotation about the x-axis in units of degrees or radians. This axis is also known as the yaw axis.

  • y_rot – The rotation about the y-axis in units of degrees or radians. This axis is also known as the pitch axis.

  • z_rot – The rotation about the z-axis in units of degrees or radians. This axis is also known as the roll axis.

  • order – The order of rotation as a string. Defaulted to ‘ZYX’, which is the traditional order for aircraft rotations. Currently this function only support ‘XYZ’, ‘XZY’, ‘YXZ’, ‘YZX’, ‘ZXY’, and ‘ZYX’.

  • deg – True if the angles x_rot, y_rot, and z_rot are in units of degrees, False for radians. Defaulted to False

  • extrinsic – True if the direction cosine matrix is to be calculated as an extrinsic rotation matrix, False for an intrinsic matrix. Defaulted to False.

This function will return a direction cosine matrix according to the Euler angles fed to the function. This function solves and multiplies the following rotation matrices where \(\theta\), \(\phi\), and \(\psi\) corresponse to roll, pitch, and yaw rotations.

\[\begin{split}\begin{align} R_x = \begin{bmatrix} 1 & 0 & 0 \\ 0 & cos\left(\theta\right) & sin\left(\theta\right) \\ 0 & -sin\left(\theta\right) & cos\left(\theta\right) \\ \end{bmatrix} \\ \\ R_y = \begin{bmatrix} cos\left(\phi\right) & 0 & -sin\left(\phi\right) \\ 0 & 1 & 0 \\ sin\left(\phi\right) & 0 & cos\left(\phi\right) \end{bmatrix} \\ \\ R_z = \begin{bmatrix} cos\left(\psi\right) & sin\left(\psi\right) & 0 \\ -sin\left(\psi\right) & cos\left(\psi\right) & 0 \\ 0 & 0 & 1 \\ \end{bmatrix} \end{align}\end{split}\]

As an example, if you were conducting a rotation about ‘XZY’, the multiplication would occur as such to obtain the intrinsic matrix

\[R_i = R_x\left(\theta\right)R_z\left(\psi\right)R_y\left(\phi\right)\]

The extrinsic matrix is determined by taking the transpose of the intrinstic matrix.

\[R_t = R_i^T\]

Code examples

from geo_py.rotations import rotation_matrix

pitch = 0.83  # radians
roll = 0.21  # radians
yaw = 1.18  # radians

# rotate about the ZYX axis, wich is default to get intrinsic matrix
dcm = rotation_matrix(yaw, pitch, roll)
print(dcm)
>>> [[ 0.25707693, -0.62399419, 0.73793137],
     [0.96289059, 0.23032494, -0.14068453],
     [-0.08217767, 0.74671392, 0.66004936]]

from geo_py.rotations import rotation_matrix

# Find the extrinsic matrix with inputs in degrees, rotated against xyz
pitch = 22.1  # degrees
roll = 8.75  # degrees
yaw = 12.8  # degrees

geo_py.rotations.dcm_euler_angles(dcm: ndarray, order: str = 'ZYX', deg: bool = False, extrinsic: bool = False) → Tuple[float, float, float][source]¶
Parameters:

  • dcm – A direction cosine matrix of order 3x3

  • order – The order of rotation as a string. Defaulted to ‘ZYX’, which is the traditional order for aircraft rotations. Currently this function only support ‘XYZ’, ‘XZY’, ‘YXZ’, ‘YZX’, ‘ZXY’, and ‘ZYX’.

  • deg – True if the angles x_rot, y_rot, and z_rot are in units of degrees, False for radians. Defaulted to False

  • extrinsic – True if the direction cosine matrix is to be calculated as an extrinsic rotation matrix, False for an intrinsic matrix. Defaulted to False.

This function will determine hte Euler angles associated with a direction cosine matrix assuming knowledge of the rotation order. Assuming, a rotation in the ‘XYZ’ orientations, the Euler angles can be determined from the following formula where \(R_{x,y}\) reprsents an attribute of the rotation matrix from postion \(x\), and \(y\).

\[\begin{split}\begin{align} \alpha=arctan2\left(\frac{R_{32}}{R_{22}}\right) \\ \beta=arcsin\left(-R_{12}\right) \\ \gamma=arctan2\left(\frac{R_{13}}{R_{11}}\right) \end{align}\end{split}\]

However, the exact nature of the equations change as a function of the orientation angle. A further description of the applicable equations can be found here rotations.

Code Example

from geo_py.rotations import rotation_matrix, dcm_euler_angles

# Find the extrinsic matrix with inputs in degrees, rotated against xyz
pitch = 22.1  # degrees
roll = 8.75  # degrees
yaw = 12.8  # degrees
dcm = rotation_matrix(ya, pitch, roll, order="XYZ", deg=True,
                      extrinsic=False)
yaw, pitch, roll = dcm_euler_angles(dcm, order="XYZ", deg=True)
print(yaw, pitch, roll)
>>> 12.8, 22.1000,8.75
